TSID 特点

  • 时间性：TSID 以时间戳为基础，在同一毫秒内生成的 TSID 保持单调递增，可以按字典序排序，便于数据库索引和查询。使用毫秒级时间戳，时间分辨率为4096 ID/毫秒/节点
  • 紧凑格式：13 个字符的字符串表示，比 UUID 更短，更易于使用和存储。可以表示为 64 位整数，便于在各种数据库和系统中使用
  • 高性能：生成速度快，无需网络或协调，适合高并发系统
  • 兼容性好：与 Java 的 tsid-creator 库完全兼容，支持跨语言使用
  • 无需中央服务器：可以在分布式系统中独立生成，无需协调
  • 自定义纪元：可以设置自定义的起始时间（纪元），延长 ID 的可用时间范围
  • URL 安全：使用 Base32 编码，所有字符都可在 URL 中安全使用
